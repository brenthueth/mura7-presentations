What is ORM?

  • Object Relational Mapping
  • Fancy phrase for "Write objects instead of SQL"
  • You write objects, objects write the SQL for you
  • Not specific to Mura or even ColdFusion
    • Mura ORM = Mura
    • CF ORM = ColdFusion
    • Hibernate = Java
    • Entity Framework = C#
    • etc

Code Time!

What's the catch with ORM?

  • Different way of thinking
  • Debugging is different
  • Can't run queries in SQL IDE
  • Everything is "select * from table"
  • (Same problems with any ORM library, not specific to Mura ORM)

Now, onward to the Feed API...

Feed API Highlights

  • Much easier syntax
  • Less verbose
  • Reads like plain English, not terse code
  • Similar to OO, jQuery, etc

Code Time!

Other Feed API Tips

  • getQuery() for CFQuery return type
    • maxItems( 0 ) to get all records
  • getIterator() to get Mura Beans
    • Page: getTitle(), getSummary()
    • Also returns extended attributes, categories, other related Mura items
    • With no extra work on your end
    • setNextN() instead of maxItems() to return all records

Other Feed API Tips

  • Ties into MXP: sort( 'mxpRelevance' )
  • Client-side option also available via JavaScript
    • JSON API & Mura ORM Via The Magic Of DataTables, Grant Shepert

What's the catch with Feed API?

  • Remember maxItems( 0 ) to get back all records
  • Builds upon Mura ORM, same behavior/performance
    • Select * from table
    • But you're no longer writing SQL!
    • If performance is truly an issue
      • Run a profiler, find the REAL problem
      • Caching options in CF/Lucee and Mura
      • Can mix and match ORM + real SQL if need be
