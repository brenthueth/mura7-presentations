Content Types

  • Custom layout markup for subtypes

Models

  • Reusble CFCs to store business logic
  • Same as "models" or "service layers" in any OO app
  • /models folder at site, theme, display_object or plugin level
  • CFCs get auto-wired by Mura
  • $.getBean( "your CFC name" )

Display Objects

  • display_objects folder
  • site, theme and plugin level
  • needs index.cfm
  • config.xml.cfm to make it drag/droppable
    • configurator.cfm to pass options into objectParams
  • Can have their own /model folder for Services
  • objectParams struct for passing in arguments
    • Like "attributes" for a Custom Tag

m() scope

  • Same as $() but on the client side
  • Replacement for jQuery
  • Does some same things as jQuery
  • select DOM elements, register events (onclick, onsubmit, etc)
  • Mura core uses this instead of jQuery
  • But you can still add jQuery on your site too
  • Can mix and match m() and jQuery

m() scope (cont)

  • Includes functions for maniuplating Display Objects
